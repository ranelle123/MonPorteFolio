Gestion Ligue Sportive

Architecture Java • Normalisation SQL • Versioning Git

Présentation du Projet

Réalisé sous Eclipse, ce projet a pour but de gérer le cycle de vie des employés au sein de différentes ligues sportives. L'enjeu était de créer une application robuste capable de gérer à la fois une logique métier complexe en Java et une persistance de données cohérente en SQL.

Stack Technique

  • Langage : Java (JDK 17+)
  • IDE : Eclipse IDE
  • Base de données : MySQL
  • Versionning : Git (Workflow basé sur des commits atomiques)

Défis Techniques & Solutions

1. Architecture de la Base de Données (MCD vers SQL)

L'un des points clés a été la modélisation de l'association "Appartenir".

Problématique : Initialement conçue comme une simple relation 1,N, j'ai dû faire évoluer le schéma pour intégrer une table de jointure.

Solution : Ce choix permet de conserver l'historique des dates d'arrivée et de départ d'un employé sans écraser ses données précédentes, respectant ainsi les principes de normalisation de la base de données.
Version 1 (Problématique)
Employe ──(1,N)── Ligue
Perte d'historique si départ
↓ Refactoring ↓
Version 2 (Solution)
Employe ──Appartenir── Ligue
Champs : dateArrivee, dateDepart
Conservation de l'historique

2. Logique Algorithmique (Traitement des chaînes)

Pour le module de recherche, j'ai implémenté un algorithme de vérification de palindromes optimisé.

Approche : Utilisation de la méthode charAt() avec une stratégie à deux pointeurs (Two Pointers).

Optimisation : La complexité est de O(n/2), car l'algorithme s'arrête dès qu'une différence est détectée, économisant ainsi des ressources processeur par rapport à une inversion complète de la chaîne.

3. Expérience Utilisateur en CLI

L'interface en ligne de commande a été pensée pour être sécurisée :

Flux de contrôle : L'utilisateur doit obligatoirement sélectionner et valider un employé par son ID avant de pouvoir accéder aux options de modification ou de suppression. Cela évite les erreurs de manipulation accidentelle de données.

Algorithmes de Recherche

Optimisation • Complexité • Tests Unitaires

Objectif du Module

Ce projet académique avait pour but d'explorer les techniques d'optimisation algorithmique en Java, avec un focus particulier sur le traitement de chaînes de caractères.

Implémentation Technique

Algorithme Two Pointers pour Palindromes

Plutôt que d'inverser la chaîne complète (O(n) en mémoire supplémentaire), j'ai opté pour une vérification in-place avec deux pointeurs.

Avantage : Complexité temporelle O(n/2) avec arrêt anticipé dès détection d'une non-correspondance.
Complexité spatiale : O(1) - pas de structure de données auxiliaire.
public class PalindromeChecker { public static boolean isPalindrome(String str) { // Normalisation (suppression espaces, minuscules) str = str.replaceAll("\\s+", "").toLowerCase(); int left = 0; int right = str.length() - 1; while (left < right) { if (str.charAt(left) != str.charAt(right)) { return false; // Arrêt anticipé } left++; right--; } return true; } }

Tests Unitaires (JUnit)

@Test public void testPalindromeCasNominal() { assertTrue(isPalindrome("kayak")); } @Test public void testPalindromeAvecEspaces() { assertTrue(isPalindrome("A man a plan a canal Panama")); } @Test public void testChaineVide() { assertFalse(isPalindrome("")); }

Analyse de Performance

Comparaison empirique avec l'approche naïve (inversion complète + comparaison) :

  • String de 1000 caractères : Gain de ~40% en temps d'exécution
  • String de 10000 caractères : Gain de ~60% (arrêt anticipé fréquent)
